# CLAUDE.md — EverMem Async

## Design Context

### Users

开发者，主要使用 AI 编程 CLI 工具（Claude Code、Codex、Kimi、Qwen Code）。他们安装 evermem-async 是为了让记忆系统"自动在后台工作"——大多数时候不需要关注 UI。偶尔打开 Web UI 有两种场景：（1）快速确认 daemon 在跑、同步正常；（2）主动搜索历史记忆 / 调试某个 agent 有没有同步成功。UI 必须同时服务好这两种模式：静默时一眼即懂，深入时层次清晰。

### Brand Personality

**优雅 · 现代 · 灵动**

参照系：Raycast、Notion、Linear 的交叉点——有工具的精准，有产品的设计感，有自己的记忆点。

- 不是冷酷的终端风，也不是消费级 App 的甜腻感
- 深色底 + 紫色主调，传递 AI / 记忆 / 技术的气质
- 克制的动效：有呼吸感，不抢戏

三词口诀：**沉稳 · 流畅 · 有记忆点**

### Aesthetic Direction

**已确定（来自代码）**
- 主背景：`#0f0f13`（接近纯黑，有蓝紫冷调）
- 卡片层：`#1a1a22`
- 边框：`#2a2a38`
- 主强调色：`#7c6af7`（中饱和紫，既有技术感又不过于刺眼）
- 文字：`#e8e8f0`（暖白，不刺眼）
- 辅助文字：`#6b6b8a`

**方向调整（来自本次对话）**
- 微交互优先：状态指示灯要有呼吸动画，按钮要有微反馈，Tab 切换要有淡入
- 卡片圆润：`border-radius: 12px+`，避免方块感
- 渐变/毛玻璃：Header 已有 `backdrop-filter: blur`，可适度扩展到卡片高亮状态
- 状态颜色语义明确：绿色=运行/成功，红色=错误/停止，黄色=警告，紫色=品牌/交互

**反参照（不要做成）**
- 不是炫技的赛博朋克（扫光、像素网格过多）
- 不是过度极简到没有个性的 admin panel
- 不是消费级 App 的大图、渐变背景、emoji 泛滥

### Design Principles

1. **静默优先，深入可达**
   Dashboard 是"一眼即懂"的监控面板：状态、倒计时、最近活动，信息够用就收手。Config 和 Search 才是深操作区，不要把操作按钮堆在首屏。

2. **动效传递状态，而非装饰**
   只在有意义的地方加动画：daemon 运行指示灯的呼吸、数据加载的骨架屏、新记忆条目的滑入。切勿给静态内容加过场动画。

3. **紫色是唯一的"活跃信号"**
   `#7c6af7` 专用于：当前选中的 Tab、启用的 Toggle、主操作按钮。其他状态（成功=绿、错误=红）用语义色，不用紫色稀释品牌。

4. **密度与呼吸的平衡**
   开发者工具需要高信息密度，但不是终端的字符堆砌。卡片内适当留白（1rem+ padding），列表项之间有间距，标签/徽章不堆叠。

5. **中英无缝**
   语言切换不应有布局抖动。两种语言共享同一套间距和字号系统，中文字符稍大的视觉补偿已由 1.5 行高覆盖。
